# BeeEffy Design System Rule

This local rule tells agents to treat the repository root `DESIGN.md` as the source of truth for UI work.

Optional style exploration lives under `design-reference/`, but it should only be used when the user explicitly asks to try another design direction.

Use this rule when you want to:

- build a new page following the project style
- refactor an existing UI to match the repository design system
- audit components for visual drift
- keep Cursor / Windsurf / Codex / Claude style output aligned
