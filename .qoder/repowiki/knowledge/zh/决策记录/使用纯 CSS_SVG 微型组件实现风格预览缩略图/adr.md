# 使用纯 CSS/SVG 微型组件实现风格预览缩略图

_来源：976215b → 1d42e71 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
指令库（PromptLibrary）包含多种设计风格，用户需要在列表中快速识别不同风格的视觉特征。需要一种轻量、低维护成本的预览方案。

## 决策驱动
- 零网络请求依赖
- 极小的 Bundle 体积
- 避免外部资源维护成本

## 备选方案
- **预生成 WebP/JPG 截图作为缩略图** _（已否决）_ — 优点：视觉效果最真实；缺点：需要构建时工具链支持；增加项目体积；每次风格调整需重新生成截图，维护成本高
- **纯 CSS/SVG 微型组件（previewHtml）** — 优点：零网络请求；体积极小；通过内联 HTML+CSS 真实反映字体、排版、配色等核心特征；易于动态生成和维护；缺点：开发初期需要为每个风格编写微型 HTML 片段

## 决策
采用纯 CSS/SVG 微型组件方案。在 `src/data/designPrompts.ts` 中为 `DesignStyle` 增加 `previewHtml` 字段，新建 `src/modes/html/StyleThumbnail.tsx` 组件，通过 `transform: scale()` 渲染微型 HTML 片段来展示风格特征。

## 影响
显著降低了资源维护负担和加载延迟。但增加了数据定义的复杂度，需要为每个内置风格编写和维护对应的 `previewHtml` 代码片段。
