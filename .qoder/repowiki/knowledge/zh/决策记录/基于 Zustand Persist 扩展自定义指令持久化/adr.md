# 基于 Zustand Persist 扩展自定义指令持久化

_来源：976215b → 1d42e71 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
用户需要创建和管理自定义的设计指令（Custom Instructions），这些数据结构复杂且需要持久化保存，同时要与现有的内置风格体系共存。

## 决策驱动
- 架构简洁性
- 复用现有状态管理机制
- 避免额外的存储同步逻辑

## 备选方案
- **使用独立的 localStorage Key 存储自定义指令** _（已否决）_ — 优点：数据隔离清晰；缺点：增加了状态管理的复杂度；需要单独处理版本迁移和数据同步；与现有 `m2v-store` 割裂
- **直接扩展 Zustand persist store** — 优点：复用现有的 `m2v-store` 持久化机制；自动处理序列化和合并；代码结构统一；缺点：需注意 localStorage 5MB 限制（已通过限制条数和字符数缓解）

## 决策
直接扩展 `src/lib/store.ts` 中的 Zustand store，新增 `customInstructions` 数组及 CRUD actions。利用 Zustand persist 的自动合并特性处理老用户数据兼容，无需额外编写 localStorage 操作代码。

## 影响
实现了自定义指令与全局状态的无缝集成。但需严格限制单条指令大小（5000字符）和总数（50条）以防止超出浏览器存储限制。
