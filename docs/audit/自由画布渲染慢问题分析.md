# 自由画布渲染慢问题分析

> 分支：`fix/html-canvas-intermittent-slowness`
> 问题现象：自由画布模式右侧渲染区偶尔快、偶尔很久都渲染不完
> 分析日期：2026-06-15

---

## 一、渲染流程

```
用户输入 → 防抖 500ms → previewHtml() 处理 → iframe srcDoc 更新
    → 浏览器解析 HTML → 加载外部资源（字体/图片/CDN）
    → iframe onLoad 事件 → detectPages + setPages + setCurrentPage
    → 缩放逻辑重新执行（因为依赖数组包含 pages/currentPage）
    → ResizeObserver 监听内容变化 → 多次 resize 计算
    → setPreviewLoading(false) → 加载动画消失
```

---

## 二、根因分析

### 根因 1：缩放逻辑在 onLoad 后执行两次

**位置**：`src/modes/html/HtmlMode.tsx:377`

**问题**：缩放逻辑的依赖数组是 `[debouncedHtml, refreshKey, pages, currentPage]`。在 `onLoad` 回调中：
```js
setPages(detected)    // 触发第一次重新执行
setCurrentPage(0)     // 触发第二次重新执行
setPreviewLoading(false)
```

每次重新执行都会：
- 创建新的 ResizeObserver
- 创建 3 个定时器 `[0, 100, 400]`
- 观察 3 个 DOM 元素

**收益**：合并 `pages` 和 `currentPage` 的更新，减少一次缩放逻辑执行。

### 根因 2：兜底 HTML 包含外部 CDN 资源

**位置**：`src/lib/extractHtml.ts:41`

**问题**：当用户输入不是以 `<` 开头时，兜底 HTML 包含 `<script src="https://cdn.tailwindcss.com">`。这是一个外部 CDN 资源，网络波动会导致：
- 加载时间不确定（100ms ~ 10s+）
- 如果 CDN 不可用，iframe 会一直等待

**收益**：移除外部 CDN 依赖，使用内联最小样式，确保渲染时间可预测。

### 根因 3：ResizeObserver 回调未节流

**位置**：`src/modes/html/HtmlMode.tsx:345`

**问题**：`observeContent()` 中的 ResizeObserver 回调直接调用 `scheduleResize()`，没有节流。如果 HTML 内容中有动画或频繁变化的元素，会触发大量 resize 计算。

**收益**：添加 requestAnimationFrame 节流，减少不必要的 resize 计算。

---

## 三、优化方案

### 方案 1：合并 onLoad 中的状态更新

将 `setPages`、`setCurrentPage`、`setPreviewLoading` 合并为一次状态更新，减少缩放逻辑执行次数。

**风险**：低（语义不变）

### 方案 2：移除兜底 HTML 的外部 CDN 依赖

将 `<script src="https://cdn.tailwindcss.com">` 替换为内联的最小样式，确保渲染时间可预测。

**风险**：低（兜底场景本身就是简单的纯文本展示）

### 方案 3：ResizeObserver 回调节流

用 `requestAnimationFrame` 包裹 ResizeObserver 回调，确保每帧最多执行一次 resize 计算。

**风险**：低（语义不变，只是减少执行频率）
