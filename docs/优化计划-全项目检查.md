## 三、代码重复消除（中优先级）

### ~~3.1 隐藏测量容器组件化~~ — 评估后不实施

`DocumentMode` 和 `CardMode` 的测量容器虽然结构相似，但样式上下文差异大（字体类名、宽度计算、CSS 变量不同），抽取组件后 props 比原代码更复杂，得不偿失。保留各模式独立实现。

### 3.2 高度测量 Hook 化

`DocumentMode`（L68-126）和 `CardMode`（L86-138）的 `useLayoutEffect` 测量逻辑高度雷同：
- ResizeObserver + image load 事件监听
- `offsetTop + offsetHeight - lastBottom` 计算物理高度
- `setActualHeights` 差异对比避免无谓渲染

**建议**：提取为 `useBlockHeights(ref, blocks, deps)` hook 到 `src/lib/`。

### 3.3 导出流程模板

CardMode 的 `exportOne`/`exportAll`/`exportZip` 和 HtmlMode 的 `handleExport*` 系列共享相同模式：

```tsx
setExporting(true)
try { /* 截图 + 下载 */ onToast('成功') }
catch (e) { onToast(`失败：${e.message}`) }
finally { setExporting(false) }
```

**建议**：抽取 `useExportAction(asyncTask, onToast)` hook，统一 exporting 状态与错误处理。

## 四、架构优化（低优先级）

### ~~4.1 App.tsx store selector 整合~~ — 评估后不实施

Zustand v5 的原子 selector（`useStore(s => s.field)`）是官方推荐模式。将多个 selector 合并为对象 hook 会在每次渲染时创建新对象引用，导致性能退化。当前 20 个独立 selector 是正确的做法。

### 4.2 HtmlMode 拆分 — 暂缓

`HtmlMode.tsx`（562 行）功能完整，作为 IDE 式组件包含 iframe 缩放、翻页、滚动同步和导出逻辑，职责内聚。拆分为多个 hook 属过度工程化，当前阶段暂缓。

### 4.3 SettingsModal 表单状态

`SettingsModal.tsx` 使用 **11 个独立 `useState`** 管理表单字段。建议改为单个 state 对象：

```tsx
const [form, setForm] = useState({
  activeType: imageHostConfig.activeType,
  smmsToken: imageHostConfig.smms?.token || '',
  // ...
})
```
