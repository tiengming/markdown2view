# 全项目优化计划

> 基于 2026-06-11 的全量代码审查（84 个源文件），涵盖构建体积、死代码、重复逻辑、架构拆分等维度。
> 当前状态：`tsc --noEmit` 通过、45 个测试全通过、`vite build` 成功（10.7s）。

---

## 一、构建体积优化（高优先级）

构建产物有 **5 个 chunk 超过 500KB**，总计约 2.1MB 未压缩 JS。

### 1.1 云存储 SDK 动态导入

| 当前问题 | `ali-oss`（690KB）和 `cos-js-sdk-v5`（168KB）被静态引入，即使未配置图床也会加载 |
|----------|---|
| 影响文件 | `src/lib/editor/imageStorage.ts` |
| 修复方式 | 将 `import OSS from 'ali-oss'` 和 `import COS from 'cos-js-sdk-v5'` 改为函数体内 `const { default: OSS } = await import('ali-oss')` 动态导入 |
| 预期收益 | 首屏 JS 减少 ~860KB（未压缩），约 237KB（gzip） |

### 1.2 配置 manualChunks 优化分包

| 当前问题 | CodeMirror + language-data 被打入 `useEditorDocSync` chunk（648KB），与 hook 逻辑无关 |
|----------|---|
| 影响文件 | `vite.config.ts` |
| 修复方式 | 在 `build.rollupOptions.output.manualChunks` 中显式拆分：`codemirror`（编辑器）、`katex`（公式字体）、`cloud-sdk`（云存储）、`screenshot`（截图导出）|
| 预期收益 | 缓存粒度更细，未使用云存储的用户不下载对应 chunk |

### 1.3 KaTeX 字体优化（低优先级）

| 当前问题 | KaTeX 字体文件产生 40+ 个独立 chunk，总计约 700KB |
|----------|---|
| 修复方式 | 评估是否只需要常用子集（中文场景下 Fraktur/Typewriter 等极少使用），或使用 CDN 按需加载 |

## 三、代码重复消除（中优先级）

### 3.1 隐藏测量容器组件化

`DocumentMode`（L247-267）和 `CardMode`（L320-343）有几乎相同的隐藏 DOM 测量容器：

```tsx
// 两处共同模式
<div ref={measuringRef} style={{ position: 'absolute', visibility: 'hidden', top: -9999, width: ... }}>
  {blocks.map(block => <section data-block-id={block.id} dangerouslySetInnerHTML={...} />)}
</div>
```

**建议**：抽取 `<HiddenMeasureContainer>` 组件到 `src/components/editor/` 或 `src/lib/`。

### 3.2 高度测量 Hook 化

`DocumentMode`（L68-126）和 `CardMode`（L86-138）的 `useLayoutEffect` 测量逻辑高度雷同：
- ResizeObserver + image load 事件监听
- `offsetTop + offsetHeight - lastBottom` 计算物理高度
- `setActualHeights` 差异对比避免无谓渲染

**建议**：提取为 `useBlockHeights(ref, blocks, deps)` hook 到 `src/lib/`。

### 3.3 导出流程模板

CardMode 的 `exportOne`/`exportAll`/`exportZip` 和 HtmlMode 的 `handleExport*` 系列共享相同模式：

```tsx
setExporting(true)
try { /* 截图 + 下载 */ onToast('成功') }
catch (e) { onToast(`失败：${e.message}`) }
finally { setExporting(false) }
```

**建议**：抽取 `useExportAction(asyncTask, onToast)` hook，统一 exporting 状态与错误处理。

## 四、架构优化（低优先级）

### 4.1 App.tsx store selector 整合

`App.tsx` 有 **20 个** `useStore` 选择器（L36-54），建议拆为自定义 hooks：

```tsx
// 将分散的 selector 聚合为语义化 hook
function useAppModeState() {
  return {
    mode: useStore(s => s.mode),
    setMode: useStore(s => s.setMode),
    colors: useStore(s => s.colors),
    // ...
  }
}
```

### 4.2 HtmlMode 拆分

`HtmlMode.tsx`（562 行）职责过多，建议拆分为：
- `useIframeAutoScale(iframeRef, pages, currentPage)` — 自动等比缩放
- `usePageNavigation(pages, iframeRef)` — 键盘/滚轮翻页
- `useIframeScrollSync(editorRef, iframeRef)` — 编辑器-预览滚动联动
- 导出函数保持独立，不需要 hook 化

### 4.3 SettingsModal 表单状态

`SettingsModal.tsx` 使用 **12 个独立 `useState`** 管理表单字段。建议改为单个 state 对象：

```tsx
const [form, setForm] = useState({
  activeType: imageHostConfig.activeType,
  smmsToken: imageHostConfig.smms?.token || '',
  // ...
})
```
