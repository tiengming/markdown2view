# 开发经验与偏好记录

本文档用于记录项目开发过程中积累的技术经验、踩坑记录以及用户的特定偏好，供后续开发和维护参考。

## 1. 用户偏好与设计准则

- **中文优先**：规划文档、需求说明、流程说明、复盘说明等优先使用中文；代码中的必要注释使用中文。
- **简单优先 (Simplicity First)**：用解决问题的最小输出，拒绝猜测性的发散。不做过度抽象。
- **目标驱动执行**：任务执行前定义成功标准，循环迭代直到验证通过。手术刀式修改，只触碰必须改动的部分。
- **UI 审美偏好**：
  - 注重“呼吸感”，避免排版局促。例如分页图文卡片的四周留白应保持全局比例协调统一（如顶部 32px，左右 30px，底部相应留白）。
  - 各模式工具栏注重模块化，不强行把所有按钮堆叠在顶部全局导航，而是根据当前模式的上下文提供最相关的操作（如 AI 指令复制、使用指南整合到局部工具栏）。
  - 界面引导：对于如“自由画布”这种高阶无代码沙箱，需提供直观的操作步骤引导（挑选风格 -> 给大模型 -> 粘回来变现）。

## 2. 核心技术经验与踩坑记录

### 2.1 公众号渲染引擎（engine）经验

- **换行处理**：换行必须用 `<br>`，且不能落在 `<span leaf="">` 内部。公众号把带 `leaf` 属性的 span 当作原子文本节点，丢弃内部的 `<br>`。因此 `leaf()` 处理时已按 `\n` 拆成多个 leaf span，`<br>` 作为兄弟节点。不要二次包裹 `leaf()`。
- **顶格输出**：`leaf()` 会对每行 `trim()`，换行后顶格输出。`inlineFormat()` 末尾用正则让 callout/breaking 等正文支持顶格换行。
- **多行标签解析**：`<title>/<p-title>/<lead>` 等支持跨行书写，闭合检测必须**非锚定**（不限于行首），否则标签无限延长。
- **中英文排版**：自动空格用 `pangu()`，已接入 `leaf()` 与 `inlineFormat()`，保证排版规范。

### 2.2 隐藏 DOM 分页与测量（A4 与图文卡片）

- **DOM 高度测量陷阱（Margin Collapse）**：直接使用 `offsetHeight` 测量段落高度时，会丢失段落间折叠的外边距（Margin）。在精确分页时，**必须使用 `offsetTop + offsetHeight - lastBottom`** 来计算包含边距的实际物理高度，避免在底部发生溢出遮挡。
- **列表项 (li) 的分页打断**：连续的列表项默认在 Markdown 渲染中属于同一个区块。如果列表过长超过一页，会在分页算法中导致整块下移或直接溢出截断。解决方案是在传入 `parseMarkdown` 之前，将列表项强制替换为段落断开（例如将 `\n- ` 替换为 `\n\n- `），使其成为独立的 Block，从而允许在列表项之间进行分页切断。

### 2.3 编辑器（CodeMirror / @uiw）经验

- **选区背景遮挡**：`.cm-activeLine` 必须使用带透明度的颜色（如 `rgba(...)`）。CodeMirror 的选区层 `.cm-selectionLayer` `z-index` 为 -1，如果行背景不透明，会导致文字拖拽选区高亮被遮挡（选得中但看不见）。
- **组件重渲染卡顿**：CodeMirror 的 `extensions` 属性必须用 `useMemo` 缓存。如果在每次 render 时生成新数组，会触发反复 `reconfigure`，造成输入严重卡顿。
- **高度填充**：使用 `height="100%"` 时，插件会让 `.cm-scroller` 采取 `height:100%!important` 以正常充满外部弹性面板。
