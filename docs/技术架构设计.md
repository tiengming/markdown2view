# markdown2view 技术架构设计

> 本文档定义 markdown2view 项目的整体架构、模块划分与关键技术方案。
> 定位：一个**纯前端、零后端**的「Markdown / HTML 多场景渲染与导出工作台」。把同一份内容渲染成面向不同受众的成品形态（长图文、A4 正式文档、单图/多图卡片、风格化 HTML），并一键复制或导出。

## 1. 核心五层架构

系统围绕“输入、共享渲染、场景、配置、导出”五层组织，各场景共享底层的代码高亮、表格、图片、公式渲染能力：

1. **输入与 AI 指令层 (`src/components/editor/`, `src/lib/aiGuide.ts`)**：
   - Markdown 编辑 / HTML 编辑 / 对应场景的 AI 提示词指令与使用指南。
2. **共享渲染内核层 (`src/engine/`)**：
   - 从 `r-markdown` 移植升级的纯 TS 引擎。处理 Markdown 解析、通用块渲染、多行标签、中英文混排空格（pangu）、代码高亮（默认换行不横铺）、表格与公式（KaTeX）。
3. **场景适配层 (`src/modes/`)**：
   - **长图文**：公众号、知乎等长文排版，附带独立的标题/摘要提取。
   - **A4 文档**：A4 固定比例，带页眉页脚，基于隐藏 DOM 实测高度的精确自动分页。
   - **社交图文卡片**：小红书 3:4 / 9:16 等比例画布，封面图+自动多页内容图拆分，统一的留白呼吸感（四周 Padding 对齐）。
   - **自由 HTML（自由画布）**：基于 iframe 沙箱的高定网页渲染，供外部 AI 生成的复杂布局使用。
4. **配置与状态层 (`src/lib/store.ts`, `src/modes/document/documentModel.ts`)**：
   - 主题、当前输入模式、全局状态，基于 `Zustand` 与 `localStorage` 进行纯前端持久化。文档模式有独立的 `DocumentSettings` 配置。
5. **导出与复制层 (`src/lib/export/`)**：
   - 包含复制富文本/HTML、前端高 DPI 截图 `iframeToBlob`（导出 PNG/长图/批量图）、浏览器原生的 PDF 打印等统一能力。

## 2. 技术栈清单

- **核心框架**：React 18 + Vite + TypeScript
- **样式**：Tailwind CSS
- **编辑器**：CodeMirror 6 (`@uiw/react-codemirror`)
- **状态管理**：Zustand
- **数学公式**：KaTeX
- **图片导出**：modern-screenshot (iframeToBlob)
- **部署方式**：纯静态输出，可部署于 GitHub Pages / Vercel

## 3. 开源归属与参考

- **[r-markdown](https://github.com/RobocopMao/r-markdown)**：公众号渲染引擎内核、自定义语法组件、主题系统、富文本/图片复制。
- **[html-anything](https://github.com/nexu-io/open-design)**：HTML iframe 沙箱容器、`iframeToBlob` 高清导图方案、导出菜单抽象。
- **[awesome-design-md](https://github.com/VoltAgent/awesome-design-md)**：自由画布中的 Design Prompt 风格指令来源。
