# markdown2view 技术路线图

> 本文档记录当前实现状态与后续优化优先级。项目坚持纯前端、零后端；AI 工作流保持「复制指令 -> 外部 AI 生成 -> 回填系统」。

## 1. 当前已完成闭环

- 长图文模式：Markdown 编辑、主题色切换、滚动联动、标题/摘要提取、富文本复制、HTML 复制、长图导出。
- A4 文档模式：页眉页脚、隐藏 DOM 实测分页、字体/主题配置持久化、PDF 导出（纯前端绘制）。
- 小红书卡片模式：封面图、发布文案、3:4 / 9:16 内容图、隐藏 DOM 实测分页、单图导出、批量导出、ZIP 打包。
- 自由 HTML 模式：HTML 编辑、iframe 沙箱预览、Prompt 指令库、单页 PNG/PDF 导出、多页识别、键盘/点击翻页演示、屏幕自适应等比拉伸、无损 ZIP 打包。

## 2. 近期优化优先级

- [x] 1. HTML 全屏演示与适配：优化 HTML 模式的大屏阅读和演示手感，实现自适应拉伸不失真、快捷键翻页。
- [x] 2. 导出链路统一：废弃系统浏览器打印，所有模式导出使用 `modern-screenshot` 截图 + `jsPDF` 生成的高清自制渲染流。
- [x] 3. AI 提示词系统专属优化：解耦 WeChat 长图文与 A4 正式文档提示词，为 A4 定制专属排版指令（突出题注与分页约定，去除社交花哨标签），并精简优化卡片元数据指令。
- [x] 4. 首包体积控制：已按模式懒加载主要工作台组件，并将大体积示例内容改为首次进入后的异步灌入；CodeMirror 语言包已做按需加载与常用集合裁剪。
- [ ] 5. A4 文档配置增强：开放更多纸张比例和边距配置，并让内容高度由页面尺寸自动计算。
- [x] 6. HTML 沙箱安全默认值：自由 HTML 模式默认禁用 iframe 内脚本，仅在用户开启“互动脚本”后允许脚本执行。

## 2.1 继续可做的工程优化

- 使用 bundle analyzer 定位主包剩余大模块，决定是否继续拆分 KaTeX、highlight.js 或 prompt 指令库。
- 为 HTML 导出链路补浏览器级 smoke 测试：多页识别、单页 PNG、多页 ZIP、PDF 导出后的样式恢复。

## 3. 中期可选方向

- 为 HTML 模式补充更多场景指令：报告、海报、幻灯片、社媒多页图、仪表盘、简历。
- 为 A4 文档增加复杂页眉页脚模板系统。
- PWA 支持，实现真正离线可用和桌面安装。

## 4. 明确不做

- 不集成本地 CLI 或后端 AI 服务。
- 不引入服务器端渲染、数据库、登录系统。
- 不把参考项目整体搬入主应用，只移植必要的纯前端能力。
