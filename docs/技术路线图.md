# markdown2view 技术路线图

> 本文档记录当前实现状态与后续优化优先级。项目坚持纯前端、零后端；AI 工作流保持「复制指令 -> 外部 AI 生成 -> 回填系统」。项目当前以**中文优先**，暂未支持多语言国际化。

## 1. 当前已完成闭环

- 长图文模式：Markdown 编辑、主题色切换、滚动联动、标题/摘要提取、富文本复制、HTML 复制、长图导出。
- A4 文档模式：复杂页眉页脚模板系统（支持左/右分栏及 `{page}/{total}` 页码变量）、Paged.js 自动分页、字体/主题配置持久化、PDF 导出（纯前端绘制）、Word（.docx）导出（基于 docx 库的 OOXML 生成，支持公式矢量 SVG 嵌入、封面页、页眉页脚、表格跨页表头重复等）。
- 小红书卡片模式：封面图、发布文案、3:4 / 9:16 内容图、基于内容单位估算的自动分页、单图导出、批量导出、ZIP 打包。
- 自由 HTML 模式：HTML 编辑、iframe 沙箱预览、Prompt 指令库（已内置幻灯片、简历、报告、海报、仪表盘、社媒多页图等场景指令）、单页 PNG/PDF 导出、多页识别、键盘/点击翻页演示、屏幕自适应等比拉伸、无损 ZIP 打包。

## 2. 近期优化优先级

- [x] 1. HTML 全屏演示与适配：优化 HTML 模式的大屏阅读和演示手感，实现自适应拉伸不失真、快捷键翻页。
- [x] 2. A4 文档模式重构（物理分页渲染）：引入 Paged.js 进行 W3C 打印标准物理分页（隔离于 iframe 沙箱内），取代了原先极其脆弱的隐藏 DOM 高度实测切割逻辑。
- [x] 2.5. 原生文本 PDF 导出恢复与优化：重新接入并优化了浏览器原生打印另存为 PDF（配合 Paged.js 媒体规则），使得导出的 PDF 具备矢量文本可选中特性、排版绝对精准、体积极小，解决原先 jsPDF 位图模糊和体积大的痛点。
- [x] 3. AI 提示词系统专属优化：解耦 WeChat 长图文与 A4 正式文档提示词，为 A4 定制专属排版指令（突出题注与分页约定，去除社交花哨标签），并精简优化卡片元数据指令。
- [x] 4. 首包体积控制：已按模式懒加载主要工作台组件，并将大体积示例内容改为首次进入后的异步灌入；CodeMirror 语言包已做按需加载与常用集合裁剪。
- [x] 5. 全场景物理分页 `<page-break/>` 统一：在 A4 文档和卡片模式下均支持用 `<page-break/>` 进行强制强行换页，并统一在工具栏加入了快捷插入按钮（风格与主工具栏扁平化一致）。
- [x] 6. HTML 沙箱安全默认值：自由 HTML 模式默认禁用 iframe 内脚本，仅在用户开启“互动脚本”后允许脚本执行。
- [x] 7. 多卡片 ZIP 导出异步非阻塞化：重构导出层，利用 fflate 异步 Web Worker 压缩实现并发无卡顿的多高清图打包 ZIP。
- [ ] 8. 扩展文档排版样式：增加其它正式文档排版样式，例如公文排版样式等。
- [ ] 8.5. 升级 Paged.js 修复跨页吞字：待 Paged.js 0.5.0 正式版发布后升级依赖，解决 v0.4.3 段落跨页拆分时丢失 2~3 行文字的上游缺陷（[Issue #167](https://github.com/pagedjs/pagedjs/issues/167)、[PR #171](https://github.com/pagedjs/pagedjs/pull/171)）。
- [x] 9. 移动端与平板端适配：引入了成熟的响应式方案（编辑/预览标签页切换、底部抽屉式面板等），合理收纳导航栏按钮图标，确保小屏设备可用性。
- [ ] 10. 自由画布指令样式完善：逐个迭代优化 HTML 示例的真实渲染效果，使其更贴近实际使用场景的产出质量。

- [x] TypeScript Node 类型声明隔离：为 `vite.config.ts` 独立类型环境。
- [x] 全局工具栏抽取：完成 `PreviewToolbar` 抽象，解决各模式交互分裂与硬编码问题。
- [x] 提示词与指令库系统重构：各模式提示词数据解耦，并支持分类本地化持久存储。
- [x] 视觉减负：剔除风格预览时的晦涩源码，凸显一键复制 CTA。
- [x] 首屏与构建体积极限优化：
  - 云存储 SDK `ali-oss` 和 `cos-js-sdk-v5` 全部改写为纯按需加载（动态 `import`）。
  - 通过 `vite.config.ts` 的 `manualChunks` 精确分离出 `react-vendor`, `codemirror-vendor`, `engine-vendor`。
- [x] 生产环境安全与纯净构建：集成 `esbuild` 配置进行 `drop_console` 和打包精简。
- [x] blockParser 提取：将 `splitMarkdownBlocks` + `classifyBlock` 提取为 `engine/blockParser/` 公共模块，A4 文档和卡片模式共享，消除重复代码。
- [x] 冗余资源清理：剥离未使用的 `demoContent.ts`、弃用的冗余测算与导出脚本。


## 3. 中期可选方向

- 多语言适配（i18n）：为项目添加完整的多语言国际化支持，使界面文案、提示词及输出内容可适配英文等多语言场景。
- PWA 支持，实现真正离线可用和桌面安装。

## 4. 明确不做

- 不集成本地 CLI 或后端 AI 服务。
- 不引入服务器端渲染、数据库、登录系统。
- 不把参考项目整体搬入主应用，只移植必要的纯前端能力。
