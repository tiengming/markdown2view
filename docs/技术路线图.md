# markdown2view 技术路线图

> 本文档记录当前实现状态与后续优化优先级。项目坚持纯前端、零后端；AI 工作流保持「复制指令 -> 外部 AI 生成 -> 回填系统」。

## 1. 当前已完成闭环

- 长图文模式：Markdown 编辑、主题色切换、滚动联动、标题/摘要提取、富文本复制、HTML 复制、长图导出。
- A4 文档模式：页眉页脚、隐藏 DOM 实测分页、字体/主题配置持久化、PDF 导出。
- 小红书卡片模式：封面图、发布文案、3:4 / 9:16 内容图、隐藏 DOM 实测分页、单图导出、批量导出、ZIP 打包。
- 自由 HTML 模式：HTML 编辑、iframe 沙箱预览、Prompt 指令库、单页 PNG/PDF 导出、多页识别、分页导出与 ZIP 打包。

## 2. 近期优化优先级

1. 自由 HTML 指令增强：补充可导出尺寸、分页容器、资源限制、截图/PDF 兼容写法，降低外部 AI 输出不可导出的概率。
2. 导出链路回归测试：补充浏览器级验证，覆盖 HTML 多页 ZIP、卡片 ZIP、A4 PDF。
3. A4 文档配置整理：将内容高度由页面尺寸和边距自动计算，降低后续开放纸张/边距配置时的错位风险。
4. 首包体积控制：按需加载 CodeMirror 语言包，或对高亮语言做常用集合裁剪。

## 3. 中期可选方向

- 为 HTML 模式补充更多场景指令：报告、海报、幻灯片、社媒多页图、仪表盘、简历。
- 为 A4 文档开放更多纸张/边距配置，并将内容高度由页面尺寸和边距自动计算。
- 控制首包体积：按需加载 CodeMirror 语言包，或对高亮语言做常用集合裁剪。
- 增加面向导出链路的浏览器级回归测试，覆盖 HTML 多页、卡片 ZIP、A4 PDF。

## 4. 明确不做

- 不集成本地 CLI 或后端 AI 服务。
- 不引入服务器端渲染、数据库、登录系统。
- 不把参考项目整体搬入主应用，只移植必要的纯前端能力。
