# 界面风格探索（优雅 / 现代）

> 更新日期：2026-06-16
> 性质：**探索性 / 假设性**。本篇与《界面风格评估.md》不冲突——前者修一致性硬伤（必做），本篇谈「如果想更进一步」的风格演进路径，仅供决策，不实施。
> 前提约束：受 `开发经验与偏好记录.md` 既有偏好框定——避免霓虹、毛玻璃、夸张渐变、Emoji、极致动效；保证导出稳定；内容（预览产物）是产品主角，外壳要"让位"。

---

## 0. 先厘清：优雅 ≠ 现代

这两个词指向**不同的设计语言**，参照系和 token 手法分叉。混在一起谈会失焦，所以本篇分开讲。

| 维度 | 优雅 (Elegant) | 现代 (Modern) |
|------|---------------|---------------|
| 气质 | 克制、编辑性、温润 | 锐利、技术感、精确 |
| 参照 | Notion / Claude / Stripe / WIRED | Vercel / Resend / Linear / Raycast |
| 字体 | 衬线标题 + 无衬线正文 | 无衬线 + 等宽点缀 |
| 色相 | 暖中性色（stone/暖灰） | 冷中性色（zinc/近黑）或深色 |
| 线条 | 更细、更浅、柔阴影 | 更硬、精确、克制动效 |
| 密度 | 放大留白、降密度 | 可维持或略增密度 |

**对这款产品的天然适配**：它是「内容渲染导出工具」，外壳是配角。**优雅方向**更安全——它强化留白与呼吸感（你本就重视），不引入会干扰内容预览的深色 chrome。**现代方向**里真正可行的是「技术感细节」（等宽点缀、精确圆角），而非深色模式（见下文风险）。

---

## 1. 约束条件（探索的边界）

探索不得突破以下既有偏好，否则与产品定位冲突：

- ❌ 不做深色 chrome 的浮夸变体（深色会让白底内容预览在 chrome 中割裂）。
- ❌ 不做毛玻璃 / 霓虹 / 夸张渐变 / 极致动效（土味 + 导出不稳定）。
- ✅ 可动的是：字体、色相微调、圆角/阴影/线条的精度、留白节奏、克制的微动效、token 系统化。

---

## 2. 探索方向

### A. 优雅取向（推荐优先探索）

**A1. 标题引入衬线 / 编辑性字体**
- 现状：全栈系统无衬线字体（`-apple-system...`）。
- 探索：仅**应用标题与各模式工具栏的次级标题**用衬线（如 Noto Serif SC / Source Han Serif），正文与控件保持无衬线。参照 Notion（serif headings）、Claude（编辑性）。
- 代价：中（需内联字体子集，否则海外 CDN 违反你的资源原则；可与「中文字体本地化」优化项合并做）。
- 风险：低。衬线只点在标题，不碰高频控件。

**A2. 暖化中性色：slate → stone/暖灰**
- 现状：冷调 slate（偏蓝）。
- 探索：切到 Tailwind `stone`（偏暖）或定制暖灰，整体观感更「纸感、温润」，呼应 A4/文档的产品语境。参照 Notion（warm minimalism）、Claude（terracotta warmth）。
- 代价：小（全局替换色相 token，但前提是先有 token 层，见 C2）。
- 风险：低。需重新校验文字对比度仍达 WCAG AA。

**A3. 放大留白节奏，降低信息密度**
- 现状：顶栏 `h-14`、工具栏 `py-2.5`，密度偏紧。
- 探索：顶栏增至 `h-16`，工具栏纵向间距 +25%，分隔线变浅。强化你已确立的「呼吸感」偏好。
- 代价：极小。
- 风险：极低，纯增益。

**A4. 更细的分隔线 + 更柔的阴影**
- 现状：`border-slate-200`、`shadow-sm`、phone-frame 用单层硬阴影。
- 探索：分隔线降到 `slate-100/200` 半透明；阴影改多层柔投影（如 `0 1px 2px rgba(0,0,0,.04), 0 4px 12px rgba(0,0,0,.06)`），替代单层。参照 Apple（premium whitespace）、Linear（precise soft shadows）。
- 代价：极小。
- 风险：无。

---

### B. 现代取向（受约束，择可行项）

**B1. 等宽字体点缀（✅ 推荐）**
- 现状：页码、状态、计数都用比例字体。
- 探索：**仅数字/状态/页码/技术性文本**用等宽（如页码「共 12 页」、文件大小、token 计数）。参照 Resend（monospace accents）、Warp。
- 代价：小。可用系统等宽栈 `ui-monospace, 'SF Mono', Menlo`，零加载成本。
- 风险：无。技术感不浮夸，且不碰导出。

**B2. 圆角精度收敛**
- 现状：rounded-md(6px) 为主，混用 rounded/lg/md。
- 探索：收敛到一套明确的圆角尺度（如 4/8/12 三档），控件统一 6px、卡片 10px、容器 12px，消除当前零星混用。参照 Linear（precise）、Vercel（precision）。
- 代价：极小。
- 风险：无。

**B3. 克制的微动效（⚠️ 谨慎）**
- 现状：以 `transition-colors` 为主。
- 探索：仅在**主操作按钮**加入克制的位移/缩放反馈（如 `active:scale-[.98]`），不做全局动效。
- 代价：小。
- 风险：中。与「避免极致动效」偏好相邻，需严格限定范围，否则易滑向浮夸。建议保守或跳过。

**B4. 深色模式（❌ 不推荐）**
- 探索：Vercel/Resend 式纯黑 chrome。
- 风险：**高**。白底内容预览（A4 文档、长图文、卡片）是产品核心，深色 chrome 会让内容区像「亮瞎眼的白洞」割裂感强；且与你「内容是主角、外壳让位」的定位冲突。**建议排除。**

---

### C. 系统级（跨取向，无论优雅还是现代都该做）

**C1. 建立 `DESIGN.md`（✅ 强烈推荐）**
- 现状：项目根目录**没有 DESIGN.md**。讽刺的是，你把整个 `awesome-design-md`（一个围绕 DESIGN.md 概念的生态）作为参考库放进了 `.agent/skills`，却没给自己的项目写一份。
- 探索：用一份 DESIGN.md 沉淀「视觉主题 / 色板角色 / 字体规则 / 组件样式 / 布局原则 / 阴影系统 / Do & Don't / 响应式」。它既是后续 AI 代理做 UI 的 single source of truth，也强迫你把当前散落在代码里的隐性风格**显性化、可审计**。
- 代价：中（写文档），但收益是结构性的——之后任何 UI 改动（含本篇所有探索项）都有据可依。
- 风险：无。纯收益。

**C2. 设计 token 化**
- 现状：颜色散落在 Tailwind 类名（slate-200、gray-200、blue-600 等），没有集中层；主题色只有 `--accent` 是变量。
- 探索：在 `index.css` 用 Tailwind v4 `@theme` 定义语义 token（`--color-border`、`--color-surface`、`--color-text-muted`、`--shadow-soft` 等），组件引用语义名而非原始 slate 值。
- 代价：中（一次性迁移）。
- 收益：本篇 A2（暖化色相）、B2（圆角收敛）等任何色相/尺度调整，从「全局搜索替换」变成「改一个变量」。是其它探索项的**前置基建**。

---

## 3. 取舍与倾向判断

基于这款产品的特性（**内容渲染导出工具，外壳是配角**）：

- **优雅方向（A 组）天然更适配**：它强化留白与呼吸感，不引入干扰内容的视觉噪音，且与你既有偏好（呼吸感、极简高级感）同向。**A3/A4 可零风险立即提升质感。**
- **现代方向里只有 B1（等宽点缀）、B2（圆角收敛）安全**；B3 谨慎、B4 排除。
- **C1（DESIGN.md）+ C2（token 化）是元基建**——无论你最终选优雅还是现代，先做这两件事，后续所有风格演进都更可控，也顺便让那个躺在 skills 里当参考的 awesome-design-md 名副其实。

**如果只挑一组做探索**：A3（留白）+ A4（柔阴影/细线）+ B1（等宽点缀）+ B2（圆角收敛）+ C1（DESIGN.md）。这组合零高风险、纯质感增益、且不与一致性评估的结论冲突。
